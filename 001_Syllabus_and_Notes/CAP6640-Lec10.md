Lecture 10: Sequence-to-Sequence Models: Bidirectional RNNs, Attention, and Neural Machine Translation
This lecture introduces sequence-to-sequence (Seq2Seq) models as a general neural framework for mapping one sequence to another. The lecture formalizes Seq2Seq learning as conditional language modeling, where the goal is to generate an output sequence conditioned on an input sequence. This formulation underlies modern approaches to machine translation, summarization, and other structured prediction tasks.

The lecture develops Seq2Seq models through the encoder–decoder architecture, examining how recurrent neural networks encode variable-length input sequences and generate variable-length outputs. It then introduces architectural extensions, including bidirectional RNNs, stacked (multi-layer) RNNs, and attention mechanisms, which address representational and optimization limitations of basic Seq2Seq models.

Key Point: Sequence-to-sequence models generate an output sequence by conditioning a language model on an encoded representation of an input sequence.

Sequence-to-Sequence Learning
In sequence-to-sequence learning, both the input and the output are sequences of variable length. Unlike classification tasks with fixed-size outputs, Seq2Seq models must learn when to begin, continue, and stop generating tokens.

Formally, given an input sequence x = (x₁, x₂, …, xn), the model predicts an output sequence y = (y₁, y₂, …, yn) by modeling:

P(y | x) = P(y₁ | x) · P(y₂ | y₁, x) · … · P(yt | y₁, …, yt₋₁, x)

This formulation shows that the decoder functions as a language model whose predictions are conditioned on the source sequence.

Encoder–Decoder Architecture
The encoder–decoder architecture consists of two neural components:

Encoder: Processes the input sequence and produces a representation
Decoder: Generates the output sequence conditioned on that representation
The encoder reads the input sequence word by word and updates a hidden state at each time step. After the final input token, the encoder outputs a context vector representing the entire sequence.

The decoder uses this context vector to initialize its hidden state and then generates the output sequence autoregressively, feeding each predicted word back as input to the next step until an end-of-sequence token is produced.

Limitation of Fixed-Length Context
A fundamental limitation of early Seq2Seq models is that the entire input sequence is compressed into a single fixed-length vector. For long sentences, this creates an information bottleneck that degrades performance.

This limitation motivates architectural enhancements that provide richer access to the input sequence during decoding.

Bidirectional RNNs
Bidirectional RNNs process a sequence in both forward and backward directions. At each time step, the model combines information from past and future context by concatenating the hidden states from both directions.

This design improves representation quality for tasks where understanding a word depends on both preceding and following words, such as named entity recognition and machine translation.

Stacked (Multi-Layer) RNNs
Stacked RNNs consist of multiple recurrent layers stacked vertically. Each layer receives as input the hidden states produced by the previous layer.

Lower layers tend to capture local and syntactic patterns, while higher layers capture more abstract semantic and global information. In practice, Seq2Seq models typically use multiple layers in both the encoder and the decoder.

Attention Mechanisms
Attention mechanisms address the fixed-length context bottleneck by allowing the decoder to dynamically focus on different parts of the input sequence at each decoding step.

Rather than relying on a single context vector, the decoder computes attention weights over all encoder hidden states and forms a weighted combination that emphasizes relevant input tokens.

Key Point: Attention allows the decoder to selectively access different parts of the input sequence during generation.
Neural Machine Translation
Neural Machine Translation (NMT) applies Seq2Seq models to the task of translating text from one language to another. NMT replaces rule-based and statistical approaches with end-to-end neural models trained on large parallel corpora.

In NMT, the encoder processes the source sentence, and the decoder generates the target sentence conditioned on the source representation. Attention mechanisms significantly improve translation quality, especially for long sentences.

Evaluating Sequence-to-Sequence Models
Seq2Seq models are evaluated using both automatic metrics and human judgment.

Perplexity: Measures how well the model predicts sequences
BLEU: n-gram precision–based comparison to reference translations
ROUGE: Recall-oriented overlap for summarization tasks
METEOR: Accounts for synonyms and word order
BERTScore: Embedding-based semantic similarity
Human evaluation assesses fluency, coherence, and relevance, often through A/B testing or user studies.

Challenges in Seq2Seq and NMT
Data scarcity for low-resource languages
Difficulty handling very long sentences
High computational cost of training
Limited interpretability and controllability
Safety and bias concerns
Lecture Takeaway
Sequence-to-sequence models generalize language modeling to conditional generation tasks. By combining encoder–decoder architectures, recurrence, depth, and attention, Seq2Seq models form the backbone of modern neural machine translation and sequence generation systems, while also highlighting fundamental challenges in scalability, evaluation, and control.
